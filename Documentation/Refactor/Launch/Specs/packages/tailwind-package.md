## 1. Purpose

`@delightful/tailwind` is a Tailwind CSS preset that maps every semantic token from the Delightful Design System to a utility class via `var()` references. Consumers add it to their `tailwind.config.js` presets array to get Delightful-aware utility classes.

---

## 2. Inputs

| Source | Path | Description |
|--------|------|-------------|
| Tailwind emitter output | `packages/tailwind/dist/preset.js` | Generated by `emitters/tailwind.mjs` during the build. Contains the complete Tailwind preset configuration. |

---

## 3. Outputs

| File | Format | Description |
|------|--------|-------------|
| `dist/preset.js` | JavaScript | Tailwind preset object. Maps semantic token names to `var()` references in `theme.extend` (or `theme` depending on the extend-vs-replace decision). |

---

## 4. Consumer Usage

```js
// tailwind.config.js
import delightful from '@delightful/tailwind';

export default {
  presets: [delightful],
  content: ['./src/**/*.{html,js,jsx,tsx}'],
};
```

After configuration, utility classes map to design tokens:

```html
<div class="bg-page text-primary border-default">
  <button class="bg-accent-primary text-on-accent">Click</button>
</div>
```

---

## 5. Key Decisions (to be resolved)

- **Extend vs replace**: Does the preset use `theme.extend` (additive, preserves Tailwind defaults) or `theme` (replaces Tailwind defaults with Delightful tokens)? Extend is safer for adoption but means consumers get both Tailwind's `blue-500` and Delightful's `accent-primary`. Replace is cleaner but breaking for existing Tailwind projects.
- **Dark mode strategy**: Does the preset configure Tailwind's `darkMode` option? The design system uses `data-theme="dark"` on `<html>`. Tailwind needs to know whether to use `class`, `media`, or `selector` strategy. The correct setting is likely `selector` with `'[data-theme="dark"]'`.
- **Utility naming**: Do we namespace utilities (e.g., `d-bg-page`) or use direct names (`bg-page`)? Namespacing avoids collisions but is verbose.

---

## 6. Open Questions

- Tailwind v3 vs v4 compatibility? The preset format differs between versions. Do we ship two presets, or target one version?
- Should the preset include spacing, radius, and shadow token mappings in addition to colors?
- Should `@delightful/tokens` be a peer dependency (consumers must install it alongside) or should the preset be fully self-contained?
