## 1. Overview

Each platform port is generated by the emitter pipeline and published via a dedicated distribution repo. The distribution repos are thin wrappers: emitter output + README + screenshots + platform-specific metadata.

## 3. Per-Platform Details

### VS Code

**File format**: JSON color themes (`delightful-light-color-theme.json`, `delightful-dark-color-theme.json`) + VS Code extension manifest (`package.json`).

**Packaging**: `vsce package` produces a `.vsix` file.

**Submission process**:
1. Create a publisher account on marketplace.visualstudio.com.
2. Run `vsce publish` to submit to the VS Code Marketplace.
3. Optionally submit to Open VSX (`npx ovsx publish`) for VS Codium users.

**Required files**:
- `package.json` (extension manifest with `contributes.themes`)
- `themes/*.json` (light + dark color themes)
- `icon.png` (128x128 extension icon)
- `README.md` with install instructions and screenshots
- `CHANGELOG.md`

**Update process**: Bump version in `package.json`, regenerate themes from emitter, run `vsce publish`.

---

### Obsidian

**File format**: `theme.css` (CSS with `--d-*` namespaced custom properties) + `manifest.json`.

**Submission process**:
1. Create a GitHub repo named `obsidian-delightful`.
2. Add `theme.css` and `manifest.json` to the repo root.
3. Open a PR to the [obsidian-releases](https://github.com/obsidianmd/obsidian-releases) repo, adding an entry to `community-css-themes.json`.

**Required `manifest.json` fields**:
```json
{
  "name": "Delightful",
  "version": "1.0.0",
  "minAppVersion": "1.0.0",
  "author": "Kyle Snavely",
  "authorUrl": "https://github.com/kylesnav"
}
```

**Update process**: Update `theme.css` from emitter output, bump `manifest.json` version, push to repo. Community themes auto-update.

### Ghostty

**File format**: `.conf` file with palette color definitions.

**Required fields**:
```
palette = 0=#1b150f
palette = 1=#ed324b
...
foreground = #1b150f
background = #fdf8f3
cursor-color = #f600a3
```

**Submission process**: Submit theme file to the [Ghostty themes gallery](https://github.com/ghostty-org/themes) via pull request, which populates ghostty.style.

**Update process**: Regenerate from emitter, update the PR or open a new one.

---

### iTerm2

**File format**: `.itermcolors` Apple property list (XML plist) with RGB float color values.

**Color encoding**: Each color is an XML dict with `Red Component`, `Green Component`, `Blue Component`, and `Alpha Component` as float values (0.0--1.0).

**Submission process**:
1. Open a PR to [mbadolato/iTerm2-Color-Schemes](https://github.com/mbadolato/iTerm2-Color-Schemes).
2. Add `Delightful Light.itermcolors` and `Delightful Dark.itermcolors` to the `schemes/` directory.

**Important note**: The iTerm2-Color-Schemes repo is also used by Ghostty's built-in theme list. Submitting to this repo automatically adds the theme to Ghostty's built-in themes as well.

**Update process**: Regenerate `.itermcolors` from emitter, update the PR or open a new one.

---

### Starship

**File format**: TOML preset for Starship prompt configuration.

**Structure**: Starship presets define prompt segment styles using hex color references:
```toml
[character]
success_symbol = "[>](bold #22a448)"
error_symbol = "[>](bold #ed324b)"

[directory]
style = "bold #00a6c0"
```

**Submission process**: Submit preset via pull request to the [Starship presets directory](https://github.com/starship/starship/discussions) or propose for inclusion at [starship.rs/presets](https://starship.rs/presets/).

**Update process**: Regenerate TOML from emitter, update the PR or discussion.

## 4. Shared Requirements

Every distribution repo includes:
- `README.md` with platform-specific install instructions
- Light theme screenshot (1280x720, captured from `docs/preview-light.html`)
- Dark theme screenshot (1280x720, captured from `docs/preview-dark.html`)
- Version number matching the source emitter output
- License file (MIT)
