# Tailwind Emitter Specification

> Defines how `palettes/delightful.json` is transformed into a Tailwind CSS preset file (`packages/tailwind/dist/preset.js`) where every semantic token maps to a `var()` reference, giving consumers access to the full Delightful palette through Tailwind utility classes.

Cross-references: [[palette-schema]] (input format), [[css-emitter]] (generates the CSS custom properties these var() references point to), [[orchestrator]] (build pipeline).

---

## 1. File & Module

```
emitters/tailwind.mjs
```

Pure function module. No file I/O -- returns output strings for the orchestrator to write.

## 2. Interface

```js
/**
 * @param {object} palette - Parsed palette JSON (validated against schema)
 * @param {object} [options] - Emitter options
 * @returns {{ files: Array<{ path: string, content: string }>, warnings: string[] }}
 */
export function emit(palette, options = {}) {
  // ...
  return {
    files: [
      { path: 'packages/tailwind/dist/preset.js', content: '...' }
    ],
    warnings: []
  };

```

### Contract

- **Input**: The full palette object (already validated).
- **Output**: A single JavaScript preset file plus any warnings.
- **Pure function**: No side effects, no file reads, no network calls.

---

## 3. Input: Palette JSON Keys Used

| Key | Purpose |
|---|---|
| `semantic.light` | All semantic token names (keys only -- values are not used) |
| `name` | Palette name for the file header comment |
| `version` | Palette version for the file header comment |

The Tailwind emitter does NOT use the OKLCH values directly. It generates `var()` references to the CSS custom properties that the CSS emitter produces. The Tailwind preset depends on `tokens.css` being loaded in the consumer's CSS.

---

## 4. Token Mapping Strategy

Every semantic token in the palette becomes a Tailwind color utility. The mapping is 1:1: each semantic token key in `palette.semantic.light` produces a corresponding entry in the Tailwind preset's `theme.extend.colors`.

### 4.1 Naming Convention

Semantic token names are kebab-case (`accent-primary`, `bg-page`). Tailwind's color config uses nested objects for grouping:

| Semantic Token | Tailwind Key Path | Generated CSS Class Examples |
|---|---|---|
| `bg-page` | `bg.page` | `bg-bg-page` |
| `bg-surface` | `bg.surface` | `bg-bg-surface` |
| `text-primary` | `text.primary` | `text-text-primary` |
| `accent-primary` | `accent.primary` | `bg-accent-primary`, `text-accent-primary`, `border-accent-primary` |
| `accent-primary-hover` | `accent.primary-hover` | `bg-accent-primary-hover` |
| `accent-primary-subtle` | `accent.primary-subtle` | `bg-accent-primary-subtle` |
| `border-default` | `border.DEFAULT` | `border-border` |
| `border-strong` | `border.strong` | `border-border-strong` |

### 4.2 Grouping Logic

Tokens are grouped by their prefix:

| Prefix | Tailwind Group | Tokens |
|---|---|---|
| `bg-` | `bg` | page, surface, elevated, subtle, muted |
| `text-` | `text` | primary, secondary, muted, on-accent, on-gold |
| `border-` | `border` | default (as DEFAULT), strong, subtle |
| `accent-` | `accent` | primary, primary-hover, primary-subtle, primary-text, danger, danger-hover, danger-subtle, danger-text, gold, gold-hover, gold-subtle, gold-text, cyan, cyan-hover, cyan-subtle, cyan-text, green, green-hover, green-subtle, green-text, purple, purple-hover, purple-subtle, purple-text |
| `status-` | `status` | info, error, warning, success |
| `focus-` | `focus` | ring |
| `overlay-` | `overlay` | bg |
| `shadow-` | `shadow` | sm, md, lg, pink, danger, gold, cyan, green, purple |

### 5.2 Key Design Decisions

- **`var()` references, not OKLCH values**: The preset references CSS custom properties from `tokens.css`. This means theme switching (light/dark via `data-theme`) works automatically -- Tailwind utilities resolve to the current theme's values at runtime.
- **No hex values**: The preset never contains hex or OKLCH color strings. All values are `var()` references.
- **1:1 coverage**: Every semantic token in the palette has a corresponding Tailwind utility. The build should verify this.
- **Shadow tokens as boxShadow**: Shadow tokens map to Tailwind's `boxShadow` config, not `colors`, because they are composite values (offset + color).
- **Foundation tokens included**: The preset also maps foundation tokens (spacing, radius, font family) to Tailwind config keys for consistency.

## 6. 1:1 Coverage Verification

The emitter performs a coverage check: every key in `palette.semantic.light` must have a corresponding entry in the generated preset. If a semantic token is not mapped, the emitter emits a warning.

```js
// After generating the preset object:
const presetKeys = flattenPresetKeys(preset.theme.extend.colors);
const paletteKeys = Object.keys(palette.semantic.light);

for (const key of paletteKeys) {
  if (!presetKeys.includes(key)) {
    warnings.push(`TailwindEmitter: semantic token '${key}' has no Tailwind mapping`);
  }
}
```

---

## 7. Generation Logic

```
1. Read palette.semantic.light keys (not values)
2. Group tokens by prefix (bg-, text-, border-, accent-, status-, focus-, overlay-, shadow-)
3. For each group, create a nested object with var() references
4. Add foundation token references (spacing, radius, font)
5. Wrap in module.exports = { theme: { extend: { ... } } }
6. Run 1:1 coverage check
7. Return { files: [{ path, content }], warnings }
```

---

## 8. Consumer Usage

Consumers reference the generated preset directly in their Tailwind config:

```js
// tailwind.config.js
const delightful = require('./packages/tailwind/dist/preset.js');

module.exports = {
  presets: [delightful],
  // ...
};
```

**Prerequisite**: The consumer's CSS must import `tokens.css` so the `var()` references resolve.

---

## 9. Error Handling

### Warnings

| Condition | Warning |
|---|---|
| Semantic token has no Tailwind mapping | `"TailwindEmitter: semantic token '{key}' has no Tailwind mapping"` |
| Unknown token prefix encountered | `"TailwindEmitter: unknown prefix for token '{key}'"` |

### Errors (thrown)

| Condition | Error |
|---|---|
| `palette.semantic` is missing | `"TailwindEmitter: palette.semantic is missing"` |
| `palette.semantic.light` is missing | `"TailwindEmitter: palette.semantic.light is missing"` |
