# Cascade Layers Specification

> How CSS cascade layers are declared, what CSS goes in each layer, how component files declare layer membership, and how consumers import layers correctly.

Cross-references: [[tokens/token-tiers]], [[foundation/reset]], [[build-pipeline]]

---

## 1. The Layer Declaration

The layer order is declared once, at the top of the CSS entry point. It establishes priority: later layers win over earlier ones regardless of selector specificity.

```css
@layer reset, primitives, semantic, component, utilities;
```

This declaration MUST appear before any `@layer` blocks that use these names. It lives in `src/reset.css` (the first file consumers import) or in an aggregating entry point.

### Layer Priority (lowest to highest)

| Priority | Layer | Override Behavior |
|----------|-------|-------------------|
| 1 (lowest) | `reset` | Base HTML element styles — overridden by everything |
| 2 | `primitives` | Raw color values — only overridden by semantic/component/utilities |
| 3 | `semantic` | Theme-aware tokens — overridden by component and utilities |
| 4 | `component` | Foundation tokens + component styles — overridden only by utilities |
| 5 (highest) | `utilities` | Utility classes — override everything |

## 2. What Goes in Each Layer

### `@layer reset`

**File**: `src/reset.css`

Contents:
- Universal box-sizing: `*, *::before, *::after { box-sizing: border-box; }`
- Margin/padding reset on `html`, `body`, block elements
- `scroll-behavior: smooth` and `scroll-padding-top` on `html`
- `:focus-visible` outline: `2px solid var(--focus-ring)` with `outline-offset: 2px`
- Base `body` typography: `font-family: var(--font-sans)`, `font-size: var(--ui-text-base)`, `line-height: var(--leading-normal)`, `color: var(--text-primary)`, `background-color: var(--bg-page)`
- Reduced-motion gate: `@media (prefers-reduced-motion: reduce) { *, *::before, *::after { animation-duration: 0.01ms !important; animation-iteration-count: 1 !important; transition-duration: 0.01ms !important; } }`
- Image and media resets: `img, video { max-width: 100%; display: block; }`

### `@layer primitives`

**File**: `src/tokens.css` (generated by CSS emitter)

Contents:
- All 44 `--primitive-*` custom properties on `:root`
- These are raw OKLCH color values with NO semantic meaning
- Static — do not change between themes

Example:
```css
@layer primitives {
  :root {
    --primitive-neutral-100: oklch(0.960 0.010 70);
    --primitive-pink-400: oklch(0.640 0.270 350);
    /* ... all 44 primitives ... */
  }
}
```

### `@layer semantic`

**File**: `src/tokens.css` (generated, same file as primitives)

Contents:
- Light theme semantic tokens on `:root, [data-theme="light"]`
- Dark theme overrides on `[data-theme="dark"]`
- All ~52 semantic tokens: backgrounds, text, borders, accents, status, shadows, focus/overlay

Example:
```css
@layer semantic {
  :root,
  [data-theme="light"] {
    --bg-page: oklch(0.982 0.008 70);
    --accent-primary: oklch(0.640 0.270 350);
    --shadow-md: 4px 4px 0 var(--border-default);
    /* ... all light tokens ... */
  }

  [data-theme="dark"] {
    --bg-page: oklch(0.140 0.014 65);
    --accent-primary: oklch(0.700 0.230 350);
    --shadow-md: 4px 4px 0 oklch(0.92 0.010 65);
    /* ... all dark overrides ... */
  }
}
```

### `@layer component`

**Files**: `src/foundation.css` + every `src/components/*.css`

Contents:
- Foundation tokens (Tier 3): all typography, spacing, radius, control heights, motion, z-index, container width custom properties
- Component-specific tokens: button color aliases, badge spacing, toggle state colors, chart palette, code syntax colors, status aliases
- All component CSS rules (`.btn`, `.card`, `.input`, etc.)

**Every component file declares its layer membership**:
```css
@layer component {
  .btn { ... }
  .btn:hover { ... }
  .btn-primary { ... }
}
```

**Critical rule**: No component file may reference `--primitive-*` tokens. Only `--semantic-*` or other component tokens via `var()`.

### `@layer utilities`

**File**: `src/utilities.css`

Contents:
- All ~28 utility classes: `.flex`, `.gap-2`, `.mt-4`, `.text-center`, `.sr-only`, etc.
- Because this layer has highest priority, these classes ALWAYS override component styles
- No component token references needed in utilities — they use explicit values or spacing tokens

Example:
```css
@layer utilities {
  .flex { display: flex; }
  .w-full { width: 100%; }
  .sr-only {
    position: absolute;
    width: 1px;
    height: 1px;
    padding: 0;
    margin: -1px;
    overflow: hidden;
    clip: rect(0, 0, 0, 0);
    white-space: nowrap;
    border: 0;
  }
}
```

## 3. How Component CSS Files Declare Layer Membership

Every component file wraps ALL its CSS in a single `@layer component` block:

```css
/* src/components/button.css */
@layer component {
  .btn {
    /* ... */
  }

  .btn:hover {
    /* ... */
  }

  .btn-primary {
    /* ... */
  }

  .btn:disabled {
    /* ... */
  }
}
```

**Rules for component files**:
1. One component per file
2. All rules inside `@layer component { }`
3. Reference only semantic tokens (`var(--accent-primary)`) or other component tokens (`var(--btn-primary-bg)`) — never primitives
4. No `@import` statements — consumers import individual files or the bundle

## 4. How Consumers Import Layers Correctly

### Full System Import (recommended for most consumers)

```css
/* In consumer's CSS entry point */
@import url("src/tokens.css");           /* primitives + semantic */
@import url("src/reset.css");              /* reset (declares layer order) */
@import url("src/foundation.css");        /* component tier foundations */
@import url("src/components/index.css");  /* all components */
@import url("src/utilities.css");          /* utilities */
```

**Import order is critical**: The layer order declaration in `reset.css` must come before any `@layer` blocks. `tokens.css` must come before `foundation.css`.

### Selective Component Import (tree-shaking)

```css
@import url("src/tokens.css");           /* primitives + semantic */
@import url("src/reset.css");              /* reset (declares layer order) */
@import url("src/foundation.css");        /* component tier foundations */
/* Import only the components you need */
@import url("src/components/button.css");
@import url("src/components/card.css");
@import url("src/utilities.css");          /* utilities */
```

This works because each component is in `@layer component` — the layer order was already declared by `reset.css`.

### Motion Import (optional)

```css
@import url("src/motion/motion.css");       /* after components */
```

Motion CSS is optional — consumers who don't need animations skip this import.

## 5. Layer Interaction Edge Cases

### Edge Case 1: Consumer adds their own CSS

A consumer can add their own CSS without layers. Unlayered CSS has HIGHER priority than all layered CSS in the cascade. This means:

```css
/* Consumer's custom.css — NO layer declaration */
.btn {
  background: red; /* THIS WINS over @layer component */
}
```

This is intentional — consumers can override any component style by writing unlayered CSS. If they want their overrides at a specific layer level, they should declare `@layer` explicitly.

### Edge Case 2: Consumer omits utilities layer

If a consumer imports tokens + foundation + components but not utilities, the `.sr-only`, `.flex`, `.gap-*` etc. classes won't be available. This is intentional — utilities are optional. Components don't depend on them.

### Edge Case 3: Importing without `reset.css`

If a consumer skips `reset.css`, the `@layer reset, primitives, semantic, component, utilities;` declaration is never made. All subsequent `@layer` blocks are defined on-demand in the order they're encountered. This usually works but loses deterministic priority.

**Recommendation**: Always import `reset.css` first, even if you override its styles with your own reset.

### Edge Case 4: `@layer` and `@import` ordering

CSS `@import` statements must come before any other rules in a file. If a consumer tries:

```css
/* WRONG — @import after a rule */
.my-class { color: red; }
@import url("tokens.css"); /* This import is ignored */
```

Use a build tool (Vite, PostCSS, Webpack) to handle imports, or ensure all `@import` statements come first.

## 6. Layer Membership Quick Reference

| File | Layer | What's In It |
|------|-------|-------------|
| `src/reset.css` | `reset` | Box model, focus, body styles, reduced-motion gate |
| `src/tokens.css` (generated) | `primitives` | 44 `--primitive-*` OKLCH values |
| `src/tokens.css` (generated) | `semantic` | ~52 semantic tokens, light + dark |
| `src/foundation.css` | `component` | Tier 3 tokens: type scale, spacing, radius, motion, z-index |
| `src/components/*.css` | `component` | All component styles |
| `src/motion/motion.css` | `component` | All 59 keyframes + animation classes |
| `src/utilities.css` | `utilities` | ~28 utility classes |
