# Directory Structure Specification

> Every file and directory in the rebuilt Delightful Design System repo, with its purpose, whether it is authored or generated, and whether it is a consumer entry point.

Cross-references: [[build-pipeline]], [[cascade-layers]], [[tokens/palette-schema]]

---

## 1. Root Layout

```
delightful/
├── palettes/               ← Color source of truth
├── emitters/               ← Build scripts (palette → outputs)
├── src/                    ← Authored CSS/JS source files
├── docs/                   ← Assembled documentation pages
├── ports/                  ← Generated platform outputs
├── tests/                  ← Playwright test suite
├── scripts/                ← Utility scripts
├── .github/                ← CI/CD workflows
├── package.json            ← npm workspaces root
├── biome.json              ← Linter config
├── playwright.config.ts    ← Test runner config
├── CLAUDE.md               ← Repo conventions for agents
├── README.md
├── CHANGELOG.md
└── LICENSE
```

---

## 2. `palettes/` — Color Authority

```
palettes/
├── delightful.json         ← AUTHORED · COLOR AUTHORITY
└── palette.schema.json     ← AUTHORED · JSON Schema validation
```

| File | Authored/Generated | Purpose |
|------|-------------------|---------|
| `delightful.json` | Authored | The single source of truth for all color. Contains primitives, semantic mappings (light/dark), and terminal hex values. EDIT THIS to change any color. |
| `palette.schema.json` | Authored | JSON Schema (draft 2020-12) used by ajv to validate `delightful.json` before any emitter runs. |

## 3. `emitters/` — Build Pipeline

```
emitters/
├── orchestrator.mjs        ← AUTHORED · Pipeline entry point
├── validate.mjs            ← AUTHORED · Schema + WCAG validation
├── css.mjs                 ← AUTHORED · palette → tokens.css
├── vscode.mjs              ← AUTHORED · palette → VS Code themes
├── obsidian.mjs            ← AUTHORED · palette → Obsidian theme
├── ghostty.mjs             ← AUTHORED · palette → Ghostty config
├── iterm2.mjs              ← AUTHORED · palette → iTerm2 colors
├── starship.mjs            ← AUTHORED · palette → Starship TOML
├── tailwind.mjs            ← AUTHORED · palette → Tailwind preset
├── palette-loader.mjs      ← AUTHORED · Shared: read + validate palette JSON
└── oklch-utils.mjs         ← AUTHORED · Shared: OKLCH parsing, hex conversion (culori)
```

All emitters implement the same interface:
```js
emit(palette, options) → { files: [{ path, content }], warnings: [] }
```

Emitters are pure functions — no file I/O. The orchestrator handles all writes.

**Consumer entry points in `src/`**:
- `src/reset.css` — import first; contains the layer order declaration
- `src/tokens.css` — import first or alongside reset; contains all color custom properties
- `src/foundation.css` — import after tokens
- `src/components/index.css` — import for full component bundle
- `src/components/{name}.css` — import individual components
- `src/motion/motion.css` — import if animations are needed
- `src/utilities.css` — import last (highest layer priority)

## 5. `docs/` — Assembled Documentation

```
docs/
├── index.html              ← ASSEMBLED · Main showcase (all 43 components)
├── color.html              ← ASSEMBLED · Palette explorer
├── preview-light.html      ← ASSEMBLED · 1280×720 screenshot target (light)
└── preview-dark.html       ← ASSEMBLED · 1280×720 screenshot target (dark)
```

Documentation pages IMPORT source CSS/JS files — they do not define tokens or components inline. They are assembled from source, not a source themselves.

## 8. `scripts/` — Utility Scripts

```
scripts/
└── bump-version.mjs        ← AUTHORED · Atomic version bump across all packages
```

`bump-version.mjs` updates the version string in:
1. `package.json` (root)
2. `palettes/delightful.json` (the `version` field)
3. Creates an annotated git tag `v{version}`

## 9. Root Config Files

| File | Purpose |
|------|---------|
| `package.json` | Project root. Defines `scripts.build`, `scripts.test`, `scripts.lint` |
| `biome.json` | Linter + formatter config. Enforces no-primitive-reference rule in component CSS. |
| `playwright.config.ts` | Playwright config: Chromium, single worker, HTML reporter, timeout settings |
| `CLAUDE.md` | Agent instructions for this repo: architecture overview, coding conventions, what not to do |
| `.gitignore` | Ignores: `node_modules/`, `dist/`, `test-results/`, `playwright-report/`, `*.itermcolors` (optional — they're generated) |
| `LICENSE` | MIT |

## 10. Authored vs Generated Summary

| Category | Authored (edit these) | Generated (never edit) |
|----------|----------------------|----------------------|
| Color data | `palettes/delightful.json` | `src/tokens.css`, all `ports/` outputs |
| Emitters | All `emitters/*.mjs` | — |
| CSS source | `src/foundation.css`, `src/reset.css`, `src/utilities.css`, `src/motion/motion.css`, `src/components/*.css` | — |
| Documentation | `docs/*.html` (template structure) | (CSS/JS injected from src imports) |
| Tests | All `tests/*.spec.ts` | — |
| Config | `biome.json`, `playwright.config.ts`, `package.json` | — |
