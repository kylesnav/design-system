Direction Overview

Linear-Precise prioritizes developer-tool precision and maximum restraint. The palette uses a cool-neutral base with a violet accent reserved for interactive elements only. Typography does all the heavy lifting through weight, size, and tight tracking. Every pixel earns its place.

Reference: Linear's monochromatic discipline, Vercel's systematic rigor, the feeling of opening a beautifully engineered IDE for the first time.

Typography

Inter variable for all text. JetBrains Mono for code. Hierarchy established entirely through weight, size, and letter-spacing.

Precision in every pixel

Inter, 740 weight clamp(2.5rem, 2rem + 2vw, 3.5rem) line-height: 1.08 letter-spacing: -0.035em

Systems that scale with intent

Inter, 680 weight clamp(1.75rem, 1.5rem + 1vw, 2.25rem) line-height: 1.15 letter-spacing: -0.025em

Token architecture for composable theming

Inter, 620 weight clamp(1.25rem, 1.1rem + 0.5vw, 1.5rem) line-height: 1.25 letter-spacing: -0.015em

Component variants through CVA

Inter, 600 weight 1.0625rem line-height: 1.35 letter-spacing: -0.005em

A design system is not a component library. It is a shared language between design and engineering, encoded as tokens, components, and patterns that enforce consistency at scale. The best systems are invisible to the end user and unmistakable to the team building with them. Every decision about spacing, color, and type should be traceable back to a single source of truth.

Inter, 400 weight 1rem (16px) line-height: 1.6 letter-spacing: normal max-width: 600px
Small text is used for secondary information, timestamps, and metadata. It maintains readability at reduced size through careful line-height and contrast choices.
