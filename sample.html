Press/Shadow Behavior Showcase

Two-tier press system: Pounce (signature) and Sink (subdued utility)

1. Buttons

All variants use the Pounce pattern. No btn-bounce class — pure translate + shadow vanish on active.

Tier 1 — Pounce: hover lift, active translate(2px, 2px) + shadow vanish

2. Tiles

Navigation tiles with their own pounce. No hover-lift utility — it conflicted with the base tile interaction.

Tier 1 — Pounce: hover translate(-4px, -4px) + shadow-lg, active translate(2px, 2px)

3. Cards (Display)

Non-interactive display cards. No hover lift, no active press — just static containers.

No interaction — display only

4. Cards (Interactive)

Cards with card-interactive class get the full pounce behavior — lift on hover, slam on active.

Tier 1 — Pounce: hover translate(-4px, -4px) + shadow-lg, active translate(2px, 2px)

5. Cards in Grid (Dimming)

When you hover one interactive card, siblings dim (opacity + grayscale). The dimming triggers on .card-interactive:hover.

Grid dimming: :has(.card-interactive:hover) .card:not(:hover) dims

6. Motion Buttons

Buttons from delightful-motion.html — hero-cta, combo-btn, and micro-btn-press — all now use pounce.

Tier 1 — Pounce: translate(-2px, -2px) hover, translate(2px, 2px) active

7. Sink Pattern

Tier 2 — subdued press for small chrome and utility controls. Slight lift on hover, translateY(1px) on active. Intentionally different from the signature pounce.

Tier 2 — Sink: hover translateY(-1px), active translateY(1px) + shadow shrink
